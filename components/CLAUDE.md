# Public Good Design System: Components

Research ../CLAUDE.md

This directory is for components documentation:

- Framework-agnostic
- Proper HTML examples
- ARIA references
- Consistent structure

For each component index.md:

- Section "When to Use" that explains suggestions for when to use the component
- Section "Headless" that explains headless usage, such as business logic, how to test,
- Section "Styles" that explains styles usage, such as CSS classes, colors, fonts, layouts, Tailwind
- Section "Testing" that explains framework-agnostic ways to test the component, its attributes, its logic
- Section "Advice" with helpful suggestions for suitable for UI/UX designers, software developers, etc.

If the comonent uses a component composition pattern:

- Section "Composition" that explains the pattern, and the relationships among related components
- Examples are *List *ListItem, \*T
